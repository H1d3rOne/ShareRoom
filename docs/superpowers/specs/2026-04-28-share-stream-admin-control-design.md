# 共享流重复展示与管理员全局控制设计

## 背景
当前实时视频共享存在两个问题：
1. 其他成员会同时在共享舞台和成员视频宫格里看到同一条共享流，形成重复画面。
2. 非共享者管理员对实时视频执行全局播放/暂停/拖动后，后续全局同步不稳定。

## 设计
- 对“成员视频宫格”与“共享舞台视频”做职责分离：共享舞台优先消费共享流；成员视频宫格只显示非共享流（如摄像头流）。当某成员当前正在提供共享流且没有额外独立视频流时，宫格不再回退显示该共享流。
- 对实时视频共享保留“管理员可发起全局控制”的权限，但把持续同步源固定为共享者本地视频源。管理员发起的 play/pause/seek/mute 仍被接受并广播；服务端对实时视频流的 canonical sync.controllerId 维持为共享者 ownerId，避免心跳源漂移到订阅端。
- share-control 对所有房间成员（含发送者）回放 canonical 同步状态，确保管理员本地 optimistic 状态被服务端权威状态纠正。

## 验证
- 回归测试覆盖：
  - 共享流不应再次进入成员视频宫格。
  - 管理员控制实时视频流后，服务端下发的 controllerId 仍为共享者。
  - 全量 `npm test` 与 `npm run build` 通过。
