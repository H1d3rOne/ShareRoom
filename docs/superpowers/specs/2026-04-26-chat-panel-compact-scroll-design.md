# 房间聊天窗口滚动与紧凑布局设计

日期：2026-04-26
文件：`pages/room/room.vue`

## 目标

对房间页右侧聊天区域做一次小范围样式优化，满足以下交互与视觉要求：

1. 聊天消息增多时，聊天窗口整体默认高度保持不变。
2. 消息列表区域内部出现纵向滚动，不把输入框向下挤走。
3. 聊天消息字体略微变小，整体更紧凑。
4. 聊天输入框左侧的 3 个设备按钮间距缩小，布局更紧凑。
5. 在不破坏现有结构的前提下，让聊天输入框获得更宽的可用空间。

## 现状

当前聊天区位于 `pages/room/room.vue` 的右侧 `chat-panel` 中，结构已经具备：

- `chat-panel`：纵向 flex 容器
- `message-list`：`flex: 1` 且 `overflow-y: auto`
- `chat-input`：位于底部，使用三列 grid（设备按钮 / 输入框 / 发送按钮）

现有结构基本正确，但视觉上还不够紧凑，且需要进一步稳定聊天区的空间分配，让“消息区滚动、输入区固定”的体验更明确。

## 方案

采用“局部样式微调，不改 DOM 结构”的方案。

### 1. 聊天窗口滚动行为

保持 `chat-panel` 为纵向布局容器，继续让 `message-list` 作为唯一可滚动区域。

调整原则：

- `chat-panel` 保持默认高度，不随消息条数增长而变高。
- `message-list` 保持 `flex: 1`、`min-height: 0`、`overflow-y: auto`。
- `chat-input` 固定在聊天面板底部，不参与滚动。
- 只优化高度分配和边距，让滚动行为更稳定、更明显。

### 2. 消息字体缩小

对聊天消息做轻量缩放：

- 普通聊天消息发送者名称略缩小。
- 普通聊天消息正文略缩小。
- 系统消息字体也同步缩小一点。
- 行高保持舒适，避免因缩小字体导致阅读发紧。

目标是“更紧凑”，不是“更密更挤”。

### 3. 输入区更紧凑

对 `chat-input` 做局部压缩：

- 左侧 `chat-device-actions` 的 `gap` 缩小。
- 三个 `device-toggle` 按钮可保持现有尺寸或只做非常轻微缩小，以优先保证点击手感。
- `chat-input` 外层内边距略微优化，减少无效留白。

### 4. 输入框变宽

在不改变功能结构的前提下，把更多横向空间让给输入框：

- 收紧左侧按钮组占用空间。
- 发送按钮保持在右侧，但不额外扩张。
- 输入框列继续使用 `minmax(0, 1fr)` 并通过整体间距压缩获得更宽的实际可用宽度。

## 不做的内容

本次不包含以下改动：

- 不改聊天消息的数据结构或发送逻辑。
- 不新增“回到底部”按钮、未读提示、吸底策略等交互。
- 不重做聊天面板视觉风格。
- 不调整房间成员面板的功能结构。
- 不修改移动端单列布局逻辑，只保证现有响应式行为不被破坏。

## 验收标准

完成后应满足：

1. 当消息数量增加时，`chat-panel` 高度看起来保持稳定。
2. 消息区域可独立滚动，输入区始终固定在底部可见。
3. 聊天消息文字比当前略小，但仍清晰易读。
4. 三个设备按钮之间的空隙比当前更小，视觉更紧凑。
5. 输入框横向宽度比当前更宽，输入区域更舒展。
6. 桌面端和当前移动端响应式布局不出现明显错位。
